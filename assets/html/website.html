Shoe Shop

The Product

Shoe shop sells shoes under two primary categories (e.g., Casual and Formal footwear). The website allows users to browse these categories, add shoes to their cart, and make purchases directly through the platform.

The Problem

There is currently no dedicated online platform for buying footwear, making it difficult for customers to find and compare shoe options—especially for casual and formal shoes. Shoppers often rely on scattered social media posts or visit physical stores, which lack transparency in pricing and limited variety. This creates a frustrating experience for users who want convenience, clear information, and options tailored to their needs.

The Goal

Develop a more intuitive and engaging shoe shopping web application that allows users to browse by category, confidently choose the right shoe (size, color), and complete purchases with ease. The platform should aim to reduce cart abandonment and improve user satisfaction through a seamless shopping journey.

User Research: Pain Points





Limited Product Variety

  • Customers feel restricted by only having two categories, making it harder to filter shoes by activity (e.g., casual, sports, formal) or other preferences like color and size..

Size Uncertainty

  • Many users are unsure if their selected size will fit, and there’s no guidance or return policy mentioned clearly.

Lack of Trust Signals

  • New users are hesitant to purchase due to missing product reviews, return policies, or user ratings.

User Journey

Paper Wireframe

The sketches represent the foundational structure and layout of the web application. They emphasize core components such as the navigation bar, homepage content sections, product grids, and interactive elements like search and filters. These early drafts helped visualize how users would interact with the platform across different screen sizes before moving into high-fidelity designs.

Digital Wireframe

The digital wireframe presents a more refined layout of the web application, highlighting key interface elements such as the header, navigation bar, product display sections, and user interaction flows. It offers a clearer visualization of content hierarchy and functionality, setting the foundation for responsive design and seamless user experiences across desktop and tablet views.

Low-Fidelity Prototype

The low-fidelity prototype offers a foundational layout of the web application, emphasizing the primary structure and essential user interactions. It serves as an early-stage visual guide to test navigation flow, page organization, and key functionalities before moving into more detailed and styled designs.

Usability Study

The usability study focuses on assessing the web application's user-friendliness, identifying areas where users face difficulties, and collecting feedback to guide design improvements. It helps ensure the interface is intuitive, efficient, and aligned with user expectations.

Refining The Design : Mockups

Based on user feedback, several important enhancements were implemented to improve the overall web app experience. For example, detailed product information was added to help users better understand the features of each shoe.

For the ordering process, a new feature was added that allows users to customize the quantity of shoes they want to purchase and easily remove items from the cart. These improvements address user needs directly, making the app more intuitive and informative.

High-Fidelity Prototype

The high-fidelity prototype represents the final visual design of the Shoe Shop Web Application, showcasing the user interface, detailed interactions, and overall user experience.

Accessbility Consideration

High Contrast & Legibility

Ensure text elements, such as plant names, prices, and care tips, have sufficient contrast against background colors and use legible, scalable fonts.

Why: Some users may struggle to read low-contrast text or decorative fonts. High contrast improves readability for everyone.

Alt Text & Labels

Ensure all images, icons, and buttons include alt text or ARIA labels for screen reader accessibility.

Why: Visually impaired users rely on screen readers to understand images and UI elements.

Conclusion

What was learned: Throughout the design process, several important insights were uncovered. User research and usability testing emphasized the need for intuitive navigation and a visually engaging layout tailored for a Shoe Shop web application. Iterative testing and feedback helped refine core features to meet user expectations—such as browsing footwear by type, viewing detailed product descriptions, and completing secure checkouts. Additionally, accessibility and customization options like filtering by size, color, and purpose (e.g., casual or sports) proved crucial in enhancing the overall shopping experience.

Next steps: Moving forward, the focus will be on developing the high-fidelity prototype into a fully functional Shoe Shop web application. Collaboration with developers will be key to ensuring the design is accurately implemented for real-world use. Continued user testing will guide future iterations and improvements. Potential upcoming features include loyalty rewards, size-fit guides, real-time order tracking, and user account personalization to increase user engagement and satisfaction.
